# 课程互动管理系统架构说明

## 1. 总体架构

系统采用单仓多应用架构：

```text
apps/
  server/   REST API、认证、权限、日志、SQL 持久化
  web/      面向桌面/平板的教学管理 Web 客户端
  mobile/   面向学生/教师/教务员的 Expo 移动客户端
packages/
  config/   共享 tsconfig / lint / build 配置
  types/    DTO、枚举、接口、角色与状态定义
  shared/   Zod 校验、常量、领域工具、API 客户端
  ui/       设计 tokens、共享视觉规范、可复用展示组件
docs/
tests/
data/
logs/
```

## 2. 分层原则

### 2.1 后端

- Controller/Route：请求解析、参数校验、权限入口
- Service：业务规则、状态流转、事务编排
- Repository：SQL 访问、实体映射、分页与筛选
- Validation/Schema：请求体、查询参数、响应结构校验
- Cross-cutting：日志、错误处理、认证、配置、审计

### 2.2 前端与移动端

- App shell：路由、认证守卫、布局框架
- Features：按业务能力拆分，如 `auth`、`courses`、`assignments`
- Shared UI：表单、表格、状态面板、空状态、角色徽标、时间轴
- Data layer：统一 API client + TanStack Query

## 3. 角色与权限模型

### 3.1 学生

- 注册、登录、找回密码、注销
- 查看并加入课程
- 查看所属课程作业
- 提交/修改自己的作业答案
- 查看成绩与教师评语
- 发布、修改、删除自己的问题/反馈
- 查看教师回复

### 3.2 教师

- 登录、维护个人资料
- 查看自己负责的课程
- 发布、修改、取消作业
- 查看作业提交列表
- 批改学生答案
- 查看并回复问题/反馈
- 查看课程互动统计

### 3.3 教务员

- 登录、维护个人资料
- 创建、维护、删除课程
- 查看课程、用户、反馈总览
- 管理教师/学生账号的启停与初始化
- 处理课程异常状态

## 4. 同步与一致性策略

### 4.1 单一事实源

- 后端数据库为唯一事实源。
- Web 和移动端不保存独立业务副本，只缓存查询结果。

### 4.2 刷新策略

- 所有写操作成功后，客户端主动失效相关查询并立即重新拉取。
- 关键列表启用短周期轮询和前台聚焦自动刷新。
- 详情页提供显式下拉刷新/手动刷新。

### 4.3 缓存策略

- 课程列表、作业列表、反馈列表使用短 TTL。
- 用户个人资料和角色上下文使用较长 TTL，但在资料更新后立即失效。
- 过期数据在 UI 中展示“上次同步时间”。

### 4.4 错误恢复

- 网络错误：展示重试按钮与最近一次成功同步时间。
- 权限错误：清理会话并回到登录页。
- 写操作失败：不保留乐观更新，直接回滚到服务端状态。

## 5. 日志与可观测性

### 5.1 后端日志

- 请求日志：方法、路径、角色、状态码、耗时、请求 ID
- 认证日志：登录成功/失败、验证码申请、密码重置、注销
- 业务日志：课程变更、作业发布/取消、批改、反馈回复
- 错误日志：异常堆栈、上下文、关联请求 ID

### 5.2 日志落盘策略

- 日志目录固定为仓库根目录 `logs/`
- `server-combined.log`：所有级别
- `server-error.log`：仅错误级别
- 测试与开发模式均保留文本日志，便于本地排错

## 6. 安全基线

- 密码哈希存储
- JWT + 服务端会话元数据
- 基于角色的路由守卫
- 所有输入经 Zod 校验
- 统一错误结构，避免静默失败
- 关键操作保留审计记录

## 7. 运行模式

### 7.1 本地开发

- Server：本地 SQL 文件数据库 + 种子数据
- Web：Vite dev server
- Mobile：Expo dev server

### 7.2 课程答辩/演示

- 使用同一后端与同一数据库
- 通过预置教师、教务员与演示课程快速进入场景
- 用日志文件与测试报告支撑答辩解释
