## 5. 核心验收流程

### WEB-ACC-01 学生注册

覆盖需求：用户注册。

前置条件：

- 未登录。
- 主学生手机号和学号未被使用。

验收步骤：

1. 打开 Web 登录页。
2. 进入“学生注册”。
3. 输入主学生手机号、学号、用户名、真实姓名、密码、确认密码。
4. 点击“获取验证码”，填写或确认系统回填验证码。
5. 点击“完成注册”。
6. 返回登录页。

预期结果：

- 注册成功后提示成功。
- 学生没有教师或教务员注册入口。
- 可使用该手机号和密码登录。

> 真实结果：学生使用手机号码注册，成功接收到验证码，并通过验证实现注册。能够成功登录系统。

### WEB-ACC-02 用户登录与角色跳转

覆盖需求：用户登录。

验收步骤：

1. 使用主学生账号登录。
2. 确认进入学生工作台，左侧包含“工作台 / 我的课程 / 我的作业 / 账号维护”。
3. 退出会话。
4. 使用教师账号登录。
5. 确认进入教师工作台，左侧包含“工作台 / 授课课程 / 作业管理 / 教学任务 / 账号维护”。
6. 退出会话。
7. 使用教务员账号登录。
8. 确认进入教务员工作台，左侧包含“工作台 / 课程运营 / 用户管理 / 课程反馈查看 / 账号维护”。

预期结果：

- 三类用户均可通过手机号和密码登录。
- 登录成功后按角色进入对应工作台。

> 真实结果：学生、教师、教务员均能成功登入系统

### WEB-ACC-03 账号维护：修改手机号与密码

覆盖需求：修改用户信息。

前置条件：

- 使用账号专项学生登录。

验收步骤：

1. 进入“账号维护”。
2. 查看个人资料区，确认姓名、手机号、学号等资料可被查看。
3. 在“修改手机号”区域输入旧手机号、新手机号。
4. 分别获取旧手机号验证码和新手机号验证码。
5. 提交手机号修改。
6. 退出后使用新手机号和原密码登录，确认登录成功。
7. 再次进入“账号维护”。
8. 输入旧密码、新密码、确认新密码，提交密码修改。
9. 退出后使用新手机号和新密码登录，确认登录成功。

预期结果：

- 手机号修改需要旧手机号和新手机号两次验证码。
- 密码修改需要旧密码、新密码、确认新密码。
- 修改后可用新凭据登录。
- 个人资料除手机号和密码外不作为本次验收的可编辑项。

> 真实结果：个人资料能正确填写并保存。“修改手机号”正确实现。“修改密码”正确实现。

### WEB-ACC-04 找回密码

覆盖需求：找回密码。

前置条件：

- 使用账号专项学生或另一个可重置账号。

验收步骤：

1. 退出登录，回到登录页。
2. 进入“忘记密码 / 找回密码”。
3. 输入手机号。
4. 获取验证码。
5. 输入新密码和确认新密码。
6. 提交重置。
7. 使用新密码登录。

预期结果：

- 密码重置成功。
- 重置后可正常登录对应角色工作台。

> 真实结果：重置密码成功，并使用新密码登入系统

### WEB-ACC-05 教务员添加课程信息

覆盖需求：添加课程信息。

前置条件：

- 使用教务员账号登录。

验收步骤：

1. 进入“课程运营”。
2. 点击“新建课程”。
3. 填写主课程信息：课程代码、课程名称、授课教师编号、学期、课程简介、授课地点、上课时间、人数上限、开始日期、结束日期。
4. 提交创建。
5. 进入新建课程详情页。

预期结果：

- 课程创建成功。
- 详情页显示课程代码、课程名称、授课教师、学期、地点、时间、状态等信息。

> 真实结果：创建成功，显示相关信息。

### WEB-ACC-06 查询课程信息

覆盖需求：查询课程信息。

验收步骤：

1. 教务员在“课程运营”中按课程名称或课程代码搜索主课程。
2. 教务员按授课教师、学期、课程状态筛选，确认主课程可被查到。
3. 教务员进入主课程详情页查看完整信息。
4. 退出后使用教师账号登录，进入“授课课程”，按课程名称或课程代码搜索，确认可看到主课程并进入课程工作区。
5. 退出后使用主学生账号登录，进入“我的课程”，按课程名称或课程代码搜索主课程并查看课程信息。

预期结果：

- 教务员、教师、学生均有 Web 入口查询课程。
- 课程名称、课程代码、授课教师、学期、状态等查询条件能支持验收人员定位目标课程。
- 搜索或筛选后仍可进入课程详情或课程工作区。

> 真实结果：教务员、教师、学生均可查看

### WEB-ACC-07 教务员修改课程信息

覆盖需求：修改课程信息。

前置条件：

- 使用教务员账号登录。
- 主课程已创建。

验收步骤：

1. 进入“课程运营”。
2. 打开主课程。
3. 进入“基础信息维护”。
4. 修改课程简介、授课地点或上课时间。
5. 点击保存。
6. 在二次确认弹窗中确认修改。
7. 返回课程概览或课程列表查看修改后的信息。

预期结果：

- 修改前出现确认提示。
- 修改后的课程信息在教务员、教师和学生可见页面保持一致。



### WEB-ACC-08 教师发布作业信息

覆盖需求：发布作业信息。

前置条件：

- 主课程已分配给预置教师。
- 使用教师账号登录。

验收步骤：

1. 进入“授课课程”。
2. 打开主课程工作区。
3. 进入“作业”Tab。
4. 填写主作业标题、描述、要求、开始时间、截止时间。
5. 提交发布。
6. 确认作业列表中出现主作业。

预期结果：

- 作业发布成功并绑定当前课程。
- 作业详情可查看标题、描述、要求、开始时间、截止时间和状态。

### WEB-ACC-09 教师查询作业信息

覆盖需求：查询作业信息。

验收步骤：

1. 教师在主课程“作业”Tab 查看主作业。
2. 教师进入左侧“作业管理”。
3. 按课程筛选主课程。
4. 按作业状态筛选。
5. 点击主作业进入课程工作区作业详情。

预期结果：

- 教师可从课程工作区和跨课程作业管理两个入口查询作业。
- 两个入口进入的是同一个作业详情上下文。

### WEB-ACC-10 教师修改作业信息

覆盖需求：修改作业信息。

前置条件：

- 主作业尚未截止。
- 主作业尚未被学生提交。

验收步骤：

1. 教师打开主课程“作业”Tab。
2. 进入主作业详情。
3. 点击“修改作业”。
4. 修改作业描述或要求。
5. 保存修改。
6. 返回作业详情确认新内容展示正确。

预期结果：

- 在截止前且无人提交时可以修改作业。
- 修改后学生端看到的是最新作业信息。

### WEB-ACC-11 学生查询作业信息并加入课程

覆盖需求：查询作业信息；同时完成后续提交前置准备。

前置条件：

- 使用主学生账号登录。
- 主课程和主作业已存在。

验收步骤：

1. 进入“我的课程”。
2. 搜索主课程。
3. 点击“加入课程”。
4. 进入课程工作区。
5. 进入“作业”Tab。
6. 查看主作业标题、描述、要求、截止时间、作业状态和我的提交状态。
7. 进入左侧“我的作业”，确认主作业也出现在跨课程作业汇总中。

预期结果：

- 学生可加入课程。
- 学生可在课程工作区和“我的作业”中查询作业。

### WEB-ACC-12 学生提交答案

覆盖需求：提交答案。

前置条件：

- 主学生已加入主课程。
- 主作业未截止。

验收步骤：

1. 学生进入主课程“作业”Tab。
2. 打开主作业详情。
3. 在提交内容中输入答案。
4. 点击“提交答案”。
5. 查看提交状态变为已提交。

预期结果：

- 未截止作业可以提交答案。
- 提交后页面展示学生自己的提交内容和提交状态。

### WEB-ACC-13 学生修改答案

覆盖需求：修改答案。

前置条件：

- 主学生已提交主作业答案。
- 主作业未截止。
- 教师尚未批改该提交。

验收步骤：

1. 学生仍停留在主作业详情页。
2. 修改提交内容。
3. 点击“修改答案”。
4. 刷新或重新进入作业详情。
5. 确认提交内容为修改后的版本。

预期结果：

- 截止前且批改前可以修改答案。
- 修改后教师侧看到的是最新提交内容。

### WEB-ACC-14 教师查询答案并批改答案

覆盖需求：查询答案；批改答案。

前置条件：

- 主学生已提交主作业答案。
- 使用教师账号登录。

验收步骤：

1. 教师进入主课程工作区。
2. 进入“提交批改”Tab。
3. 按作业筛选主作业。
4. 在提交列表中找到主学生提交。
5. 打开提交详情，查看学生姓名、学号、提交内容、提交时间。
6. 输入分数和教师批改回复。
7. 点击“提交批改”。

预期结果：

- 教师可查看学生答案。
- 批改保存成功。
- 提交状态变为已批改。

### WEB-ACC-15 学生查询答案和批改结果

覆盖需求：查询答案。

前置条件：

- 教师已批改主学生提交。

验收步骤：

1. 使用主学生账号登录。
2. 进入“我的作业”。
3. 打开主作业详情。
4. 查看提交内容、提交状态、分数、教师批改回复。

预期结果：

- 学生可查询自己的答案。
- 学生可看到是否已批改、分数和教师批改回复。

### WEB-ACC-16 学生发布问题或反馈

覆盖需求：发布问题或反馈。

前置条件：

- 主学生提交已被教师批改。

验收步骤：

1. 学生打开主作业详情。
2. 确认提交结果区下方显示“我有问题/反馈”入口。
3. 点击该入口进入反馈编辑区域。
4. 选择“问题”，输入与批改结果相关的问题内容。
5. 发布问题。
6. 再发布一条“反馈”类型内容，作为后续修改和删除用例的数据。

预期结果：

- 只有已批改提交才能作为正常入口发起问题或反馈。
- 发布成功后可在作业反馈区或作业反馈 Tab 中看到对应线程。

### WEB-ACC-17 学生和教师查看问题或反馈

覆盖需求：查看问题或反馈。

验收步骤：

1. 学生进入主课程“作业反馈”Tab。
2. 查看自己刚发布的问题和反馈。
3. 进入问题线程，查看作业上下文、反馈内容和教师回复区域。
4. 退出后使用教师账号登录。
5. 教师进入“教学任务”或主课程“作业反馈”Tab。
6. 查看主学生发布的问题或反馈。

预期结果：

- 学生可查看自己发布的问题或反馈。
- 教师可查看学生发布的问题或反馈。
- 线程展示课程、作业、学生、提交和批改上下文。

### WEB-ACC-18 学生修改问题或反馈

覆盖需求：修改问题或反馈。

前置条件：

- 主学生已发布至少一条反馈类型内容。

验收步骤：

1. 学生进入主课程“作业反馈”Tab。
2. 打开反馈类型线程。
3. 点击“修改”。
4. 修改反馈内容。
5. 保存修改。
6. 重新进入该线程，确认内容已更新。

预期结果：

- 学生可修改自己发布的问题或反馈。
- 修改后的内容在学生和教师侧均可看到。

### WEB-ACC-19 学生删除问题或反馈

覆盖需求：删除问题或反馈。

前置条件：

- 主学生已发布用于删除的反馈类型线程。

验收步骤：

1. 学生打开用于删除的反馈线程。
2. 点击“删除”。
3. 在确认提示中确认删除。
4. 返回“作业反馈”Tab。
5. 确认该反馈线程不再显示。

预期结果：

- 学生可删除自己发布的问题或反馈。
- 删除后该线程不再作为正常可访问反馈显示。

### WEB-ACC-20 教师增加回答

覆盖需求：增加回答。

前置条件：

- 主学生保留一条未删除的问题线程。
- 使用教师账号登录。

验收步骤：

1. 教师进入“教学任务”。
2. 切换到“未回答作业反馈”队列。
3. 打开主学生的问题线程。
4. 在“新增回复”中输入回答内容。
5. 点击“发布回复”。

预期结果：

- 教师可对学生问题增加回答。
- 线程状态可自然体现为已回答。

### WEB-ACC-21 学生和教师查看回答

覆盖需求：查看回答。

验收步骤：

1. 教师在当前线程中查看刚发布的回答。
2. 退出后使用主学生账号登录。
3. 学生进入主课程“作业反馈”Tab。
4. 打开问题线程。
5. 查看教师回答内容。

预期结果：

- 教师可查看自己发布的回答。
- 学生可查看教师给出的回答。

### WEB-ACC-22 教师修改回答

覆盖需求：修改回答。

前置条件：

- 教师已发布回答。

验收步骤：

1. 使用教师账号打开主学生的问题线程。
2. 点击自己回答旁的“修改”。
3. 修改回答内容。
4. 保存。
5. 刷新或重新进入线程。

预期结果：

- 教师可修改自己发布的回答。
- 学生侧能看到修改后的回答内容。

### WEB-ACC-23 教师删除回答

覆盖需求：删除回答。

前置条件：

- 教师已发布至少一条回答。

验收步骤：

1. 教师打开主学生的问题线程。
2. 点击自己回答旁的“删除”。
3. 在确认提示中确认删除。
4. 查看回答列表。

预期结果：

- 教师可删除自己发布的回答。
- 删除后该回答不再显示。

### WEB-ACC-24 教师删除或取消作业信息

覆盖需求：删除作业信息。

前置条件：

- 使用教师账号登录。
- 主课程下创建一个单独的“Web验收取消作业”。

验收步骤：

1. 教师进入主课程“作业”Tab。
2. 打开“Web验收取消作业”详情。
3. 填写取消原因。
4. 点击“取消作业”。
5. 在确认提示中确认取消。
6. 返回作业列表查看作业状态。

预期结果：

- 取消前要求填写原因。
- 确认提示明确说明会清除所有学生在该作业下的提交记录且不可恢复。
- 作业状态变为已取消。

### WEB-ACC-25 教务员删除课程信息

覆盖需求：删除课程信息。

前置条件：

- 使用教务员账号登录。
- 已创建单独的待删除课程，不使用主课程。

验收步骤：

1. 教务员进入“课程运营”。
2. 新建或打开 `WEB-DEL-0520` 待删除课程。
3. 进入“基础信息维护”。
4. 查看删除课程区域。
5. 确认页面列出关联作业数、关联提交数、关联作业反馈数、关联课程整体反馈数。
6. 点击“删除课程”。
7. 在确认提示中确认删除。
8. 返回课程运营列表，搜索该课程代码。

预期结果：

- 删除前出现确认提示。
- 确认提示说明删除后不可恢复，并列出级联影响范围。
- 删除后该课程不再出现在课程列表中。

### WEB-ACC-26 注销账号

覆盖需求：注销账号。

前置条件：

- 使用账号专项学生登录，不使用主学生账号。

验收步骤：

1. 进入“账号维护”。
2. 点击“注销账号”。
3. 在二次确认提示中确认注销。
4. 系统退出到登录页。
5. 尝试使用该账号登录。

预期结果：

- 注销前出现危险操作确认。
- 注销后账号不能继续进入系统。
- 主学生账号不受影响，可继续完成其他验收。

## 7. 可选附加验收：课程整体反馈

本节不计入核心 25 项。若答辩或评审希望展示 Web 新版保留的“课程整体反馈”能力，可按以下流程补充验收。

### WEB-EXT-01 学生提交课程整体反馈

验收步骤：

1. 主学生进入主课程工作区。
2. 打开“课程整体反馈”Tab。
3. 选择反馈维度。
4. 输入反馈内容。
5. 提交反馈。

预期结果：

- 学生可提交课程层面的反馈。
- 页面展示“我的课程整体反馈”。

### WEB-EXT-02 学生修改和删除课程整体反馈

验收步骤：

1. 学生在“课程整体反馈”Tab 找到自己提交的反馈。
2. 点击“修改”，更新内容并保存。
3. 确认内容已更新。
4. 点击“删除”，在确认提示中确认。

预期结果：

- 学生可修改自己的课程整体反馈。
- 学生可删除自己的课程整体反馈。

### WEB-EXT-03 教师查看课程整体反馈

验收步骤：

1. 教师进入主课程工作区。
2. 打开“课程整体反馈”Tab。

预期结果：

- 教师只读查看自己授课课程下的课程整体反馈。

### WEB-EXT-04 教务员查看课程整体反馈

验收步骤：

1. 教务员进入“课程反馈查看”全局入口。

3. 进入某个课程详情。
4. 打开“课程反馈查看”Tab。

预期结果：

- 教务员可全局只读查看课程整体反馈。
- 教务员也可在课程详情内只读查看该课程反馈。
