# Web 端页面清单与生图 Prompt

生成日期：2026-05-01

## 页面数量口径

当前 Web 端代码是单页 React 应用，严格路由层面只有 `App.tsx` 一个 SPA 页面。为了后续美化和生成参考图，按“验收功能 + 角色工作流 + 页面级视图”拆分为 **13 个 Web 页面级视图**。

不计入独立页面的内容：`当前进度`、`状态提示`、`通知条`、`空状态`等，它们作为各页面的辅助组件统一设计。

## 统一视觉系统 Prompt 前缀

以下每个页面 prompt 都应继承这段统一风格，保证后续生成的图片一致：

```text
为“课程互动管理系统”设计一张桌面 Web 应用界面参考图，1440x1024，中文高校课程管理后台，不是移动端，不是营销落地页。整体风格为安静、专业、现代的教学运营 SaaS：浅色背景，白色主内容区，低饱和冷灰底色，深墨色文字，主色为深青绿色，辅助色为理性蓝、琥珀色和少量玫红危险色。布局密集但清晰，左侧固定导航栏，顶部状态栏，主内容区用表格、列表、表单、分栏和状态徽标组织信息。卡片圆角不超过 8px，细边框，轻阴影，按钮带简洁线性图标。中文字体清晰易读，标题克制，不要巨大营销式 hero，不要紫色渐变，不要装饰性光球，不要夸张玻璃拟态。界面需要像真实可用的高校课程互动管理系统，而不是通用模板。
```

建议图片命名目录：`docs/web-ui-references/`

## 01. 统一身份认证页

建议文件名：`01-auth-login-reset.png`

页面用途：教师、教务员、学生通过手机号和密码登录；忘记密码时通过验证码重置密码。

核心区域：

- 顶部品牌栏：课程互动管理系统、统一身份认证。
- 左侧说明区：账号服务、教师与教务员使用分配账号、学生注册后登录、账号支持说明。
- 登录表单：手机号、密码。
- 找回密码表单：手机号、验证码、新密码、确认新密码。
- 状态提示条：登录失败、验证码发送、重置成功等消息。

按钮：

- `账号登录`
- `学生注册`
- `进入工作台`
- `获取重置验证码`
- `重置密码`

Prompt：

```text
使用统一视觉系统。绘制“统一身份认证页”：桌面 Web 登录界面，左侧是深青绿色教学服务说明面板，右侧是白色登录表单。表单顶部有“身份认证 / 账号登录”，包含手机号输入框、密码输入框、主按钮“进入工作台”。登录表单下方同页展示“忘记密码”区域，包含手机号、验证码、新密码、确认新密码，以及“获取重置验证码”“重置密码”两个按钮。顶部有系统品牌栏“课程互动管理系统”和服务状态。界面要体现高校教学平台、安全认证、清晰、正式，中文文字真实可读，按钮层级明显，避免营销海报感。
```

## 全局组件要求

后续用生成图片落地前端时，所有页面共用这些组件风格：

- 左侧导航：统一宽度，角色不同但结构一致。
- 顶部栏：当前用户、角色、服务状态、退出按钮。
- 表单控件：同一高度、同一边框、同一 focus 状态。
- 主按钮：深青绿色；次级按钮：白底细边框；危险按钮：低饱和玫红。
- 状态徽标：进行中、未开始、已完成、暂停、已提交、未提交、待批改、已批改。
- 数据列表：课程卡、作业卡、提交卡、反馈线程卡保持一致。
- 空状态：统一标题、简短说明，不使用插画堆砌。
- 响应式：优先保证 1440px 桌面验收；1024px 宽度下改为两列或单列。
