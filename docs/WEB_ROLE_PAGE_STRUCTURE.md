# Web 端角色页面结构设计

更新日期：2026-05-20

本文档记录 Web 端按角色组织的页面结构。当前覆盖公共认证、学生端、教师端和教务员端。

## 0. 公共认证与账号生命周期

公共认证页面不归属于某个角色的左侧导航，位于登录前和账号安全流程中。

### 0.1 登录

登录页支持学生、教师、教务员使用手机号和密码登录。

登录成功后，系统根据用户角色跳转到对应工作台：

- 学生 -> 学生端工作台
- 教师 -> 教师端工作台
- 教务员 -> 教务员端工作台

登录失败时，应明确提示手机号或密码错误、账号已禁用、账号已注销等状态。禁用或注销账号不能继续进入系统。

### 0.2 学生注册

学生注册页只面向学生开放。

注册流程包括：

- 输入手机号
- 获取并填写验证码
- 设置密码
- 二次确认密码
- 填写学生基础信息

教师和教务员无需自助注册。教师账号由教务员创建；教务员账号由系统管理员预置。

### 0.3 密码找回

密码找回页支持学生、教师、教务员通过注册或分配时绑定的手机号找回密码。

流程包括：

- 输入手机号
- 获取并填写验证码
- 设置新密码
- 二次确认新密码
- 重置成功后返回登录页或自动提示重新登录

### 0.4 账号维护

登录后，各角色的“账号维护”入口承载账号生命周期相关功能。

账号维护应包括：

- 查看个人资料
- 修改个人资料
- 修改手机号
- 修改密码
- 注销账号
- 退出登录

修改手机号需要旧手机号验证和新手机号验证。修改密码需要输入旧密码、新密码和新密码确认。

注销账号属于危险操作，需要二次确认。注销后该账号不可继续登录；学生账号如需再次使用系统，应重新注册。教师和教务员账号注销或禁用后的恢复应走管理流程。

## 1. 总体原则

Web 端不再直接按需求文档中的功能模块或数据库类拆分左侧导航。真实业务链路应围绕教学上下文组织：

```text
课程 -> 作业 -> 提交/批改 -> 作业问题或反馈 -> 教师答疑回复
```

因此，左侧导航只进入大的业务空间；具体操作应在课程、作业、提交这些上下文内完成。

反馈分为两类：

1. 作业反馈：绑定某次作业和批改结果，是学生围绕具体提交、分数、教师评语继续沟通的主反馈流。
2. 课程整体反馈：绑定课程反馈任务，不绑定某次作业。课程结束后自动出现，教师也可以在课程进行中手动触发。

课程整体反馈采用任务制，不是长期开放意见箱。学生提交课程整体反馈时始终可以选择匿名；匿名仅对教师匿名，系统和教务员保留审计能力。

本设计以 `doc/001 B 课程互动管理系统 需求获取报告 5.docx` 的功能范围为基础。确实偏离原始需求文字但更符合正常使用逻辑的地方，需要在本文档中显式标注为“设计解释”或“设计增强”，避免实现和验收时产生歧义。

## 2. 学生端

### 2.1 左侧导航

学生端左侧导航建议保留以下入口：

```text
工作台
我的课程
我的作业
课程反馈任务
消息
账号维护
```

入口职责：

| 入口 | 职责 |
| --- | --- |
| 工作台 | 展示学生当前最需要处理的学习待办，例如待提交作业、已批改作业提醒、待填写课程整体反馈任务。 |
| 我的课程 | 查看已加入课程和可加入课程；进入某门课程后进入课程工作区。 |
| 我的作业 | 跨课程汇总所有作业状态，点击作业后进入对应课程工作区的作业详情。 |
| 课程反馈任务 | 汇总所有课程级反馈任务，包括待填写、已提交、已过期任务。只处理课程整体反馈，不处理作业反馈。 |
| 消息 | 汇总教师回复、系统提醒、课程反馈任务提醒等通知，不作为发起反馈的主要入口。 |
| 账号维护 | 个人资料、手机号、密码、安全操作。 |

### 2.2 课程工作区

学生从“我的课程”进入某门课程后，进入课程工作区。

课程工作区顶部固定展示课程上下文：

```text
课程名称 / 授课教师 / 学期 / 上课时间地点 / 课程状态 / 当前学习进度
```

课程工作区使用以下 Tab：

```text
课程概览 / 作业 / 作业反馈 / 课程整体反馈 / 资料与通知
```

### 2.3 课程概览

课程概览用于帮助学生确认自己当前所在课程，不承担密集操作。

主要内容：

- 课程简介
- 授课教师
- 学期
- 上课时间与地点
- 课程状态
- 课程人数或已加入状态
- 最近待办摘要

### 2.4 作业

作业 Tab 是学生端主操作区。

作业列表应区分“作业状态”和“我的提交状态”，避免把课程作业本身状态与学生个人提交状态混在一起。

作业状态展示：

- 未开始
- 进行中
- 已截止
- 已完成
- 已取消

我的提交状态展示：

- 未提交
- 已提交
- 已批改
- 未完成

点击某次作业后，在同一页面完成：

- 查看作业说明、要求、开始时间、截止时间
- 提交答案
- 在允许条件下修改答案
- 在允许条件下删除答案
- 查看提交状态
- 查看分数
- 查看教师批改评语

答案删除规则：

- 学生可以删除自己尚未批改的答案。
- 删除答案需要二次确认。
- 已批改答案不允许删除。
- 作业已截止后是否允许删除答案，应与提交/修改答案的截止规则保持一致：原则上不允许删除。
- 删除后，该学生在该作业下的提交状态回到“未提交”或“未完成”。

作业详情页必须天然带有以下上下文：

```text
courseId / assignmentId / submissionId
```

### 2.5 作业反馈

作业反馈不再作为左侧独立主入口，而是课程工作区内的上下文功能。

作业反馈 Tab 按“已批改作业”列出可反馈对象。每条记录展示：

- 作业名称
- 分数
- 教师评语
- 是否已有问题或反馈
- 教师是否已回复

学生点击某条作业反馈对象后，进入该作业的反馈线程。

发起作业反馈的推荐条件：

- 学生必须已有提交记录。
- 推荐在作业已批改后才允许发起问题或反馈。
- 反馈内容必须绑定到具体 `courseId / assignmentId / submissionId`。

作业反馈线程展示：

- 学生问题或反馈内容
- 教师回复
- 回复时间
- 当前处理状态

学生可以在作业反馈线程中管理自己发布的问题或反馈：

- 查看自己发布的问题或反馈
- 修改自己发布的问题或反馈
- 删除自己发布的问题或反馈

修改或删除问题/反馈前，应校验该记录确实属于当前学生。若教师已经回答，该问题或反馈仍允许学生修改或删除，但页面应提示“该内容已有教师回答，修改或删除可能影响上下文”。第一版不做更复杂的锁定规则。

### 2.6 课程整体反馈

课程整体反馈只处理课程级反馈任务。

课程整体反馈任务来源：

1. 自动触发：课程状态变为“已结课”，或课程结束日期到达后，系统自动生成课程整体反馈任务。
2. 教师手动触发：课程进行中，教师可在课程工作区手动发起课程整体反馈任务。

课程整体反馈任务字段建议：

- 反馈标题
- 反馈说明
- 开始时间
- 截止时间
- 任务状态
- 是否已提交

学生提交课程整体反馈时：

- 始终允许勾选“匿名提交”。
- 匿名仅对教师匿名。
- 系统和教务员仍保留真实提交人，用于审计。
- 学生自己查看历史反馈时，应能看到自己提交过该反馈，并能看到是否匿名提交。

教师端展示匿名反馈时：

```text
匿名学生
```

不得展示学生姓名、学号、班级、手机号等身份信息。

教务员端审计时可以查看真实提交人。

### 2.7 资料与通知

该 Tab 作为课程附属信息区，短期可以保持轻量。

可展示：

- 课程说明
- 教师补充通知
- 课程安排变更
- 后续可扩展的课件或资料入口

## 3. 教师端

### 3.1 左侧导航

教师端左侧导航建议保留以下入口：

```text
工作台
授课课程
作业管理
教学任务
课程反馈任务
消息
账号维护
```

入口职责：

| 入口 | 职责 |
| --- | --- |
| 工作台 | 展示跨课程教学待办和摘要，例如待批改提交、未回答作业反馈、即将截止作业、进行中的课程反馈任务。 |
| 授课课程 | 查看教师本人授课课程；进入某门课程后进入课程工作区。 |
| 作业管理 | 跨课程管理作业对象，包括发布、修改、删除/取消、查询和查看提交统计。 |
| 教学任务 | 处理教师待办任务，包括待批改提交和未回答作业反馈。 |
| 课程反馈任务 | 发起、查看和管理课程整体反馈任务，查看匿名脱敏后的反馈汇总。 |
| 消息 | 聚合通知并跳转到对应业务页面，不承载业务处理状态。 |
| 账号维护 | 个人资料、手机号、密码、安全操作。 |

`作业管理` 和 `教学任务` 必须保持对象边界清晰：

- `作业管理` 面向“作业对象”，关注作业的创建、维护、状态和统计。
- `教学任务` 面向“待处理任务”，关注教师当前需要批改或回复的事项。

### 3.2 工作台

教师工作台用于跨课程展示当前最需要处理的教学事务，不放置复杂业务表单。

建议展示：

- 当前授课课程数
- 已发布作业数
- 待批改提交数
- 未回答作业反馈数
- 进行中的课程整体反馈任务
- 即将截止的作业
- 即将截止的课程反馈任务

工作台中的待办项点击后，应跳转到对应课程、作业、提交或反馈线程，并自动带入上下文。

### 3.3 授课课程与课程工作区

教师从“授课课程”进入某门课程后，进入课程工作区。

课程工作区顶部固定展示课程上下文：

```text
课程名称 / 学期 / 课程状态 / 上课时间地点 / 学生人数 / 作业数 / 待处理数
```

课程工作区使用以下 Tab：

```text
课程概览 / 作业 / 提交批改 / 作业反馈 / 课程反馈任务 / 学生名单
```

#### 3.3.1 课程概览

课程概览用于帮助教师快速理解当前课程运行状态。

主要内容：

- 课程简介
- 学期
- 上课时间与地点
- 课程状态
- 学生人数
- 最近作业
- 待批改提交摘要
- 未回答作业反馈摘要
- 课程整体反馈任务摘要

教师一般不在该页面编辑课程基础信息；课程基础信息维护仍以教务员端为主。

#### 3.3.2 作业

课程工作区的“作业” Tab 用于管理当前课程下的作业。

支持能力应与需求分析报告保持一致：

- 发布作业
- 修改作业
- 删除/取消作业
- 查询作业
- 查看提交统计

约束：

- 作业必须绑定课程。
- 从课程工作区发布作业时，自动带入当前课程。
- 教师只能为自己授课的课程发布作业。
- 修改作业遵循需求文档中的约束：教师在作业截止前可以对已发布但未被学生提交的作业进行修改。
- 删除/取消作业时需要确认，并填写必要原因；学生端应能理解作业已取消。

作业删除/取消规则：

- 需求获取文档中描述为“删除作业”，并提到清除学生提交记录。
- Web 第一版将该操作设计为“取消作业”优先，而不是常规物理删除。
- 取消后，学生端显示作业已取消和取消原因。
- 已有学生提交时，不直接抹除历史提交内容；这些提交应从有效作业统计中排除，但可在教师端和必要的审计视图中保留查看。
- 若未来必须做物理删除，应提供更高风险提示或更高权限流程，不作为教师端普通操作。

#### 3.3.3 提交批改

“提交批改” Tab 用于处理学生答案。

教师在该页面完成：

- 查看学生提交答案
- 填写分数
- 填写教师批改回复
- 保存批改结果
- 查询已批改答案

批改页面必须展示完整上下文：

```text
课程 -> 作业 -> 学生 -> 提交内容 -> 分数/批改回复
```

第一版不做以下能力：

- 打回重交
- 批改版本历史
- 单独成绩分析入口

教师可以修改已批改提交的分数和批改回复，这是为了纠正误判、漏判等实际教学场景而保留的设计增强。当前阶段只保留最新结果，不做批改版本历史。若该提交已经产生作业问题或反馈，修改时应提示教师：修改批改结果可能影响已有沟通上下文。

#### 3.3.4 作业反馈

“作业反馈” Tab 用于处理学生在作业批改后提出的问题或反馈。

列表展示：

- 课程名
- 作业名
- 学生姓名 / 学号
- 问题或反馈内容
- 发布时间
- 回答状态
- 操作入口

回答状态不新增复杂业务状态，只由回答记录自然推导：

- `未回答`：该问题或反馈下没有教师回答。
- `已回答`：该问题或反馈下已有至少一条教师回答。

反馈详情展示：

- 课程信息
- 作业信息
- 学生提交答案
- 教师批改分数与批改回复
- 学生问题或反馈
- 教师回答列表
- 新增回答表单

一个学生问题或反馈下允许教师发布多条回答。教师可以查看、修改、删除自己的回答。第一版只做按时间排列的回答列表，不做复杂楼中楼。

第一版不做：

- 标记已解决
- 重新打开
- 额外反馈状态流转

### 3.4 作业管理

“作业管理”是跨课程作业中心，适合教师快速处理所有授课课程下的作业对象。

支持能力：

- 按课程筛选作业
- 按作业状态筛选作业
- 按截止时间排序
- 发布作业
- 修改作业
- 删除/取消作业
- 查看提交统计
- 快速进入某个作业的提交批改页

从“作业管理”发布作业时，教师必须先选择课程。可选课程仅限教师本人授课课程。如果教师当前没有授课课程，发布入口应禁用，并提示联系教务员分配课程。

### 3.5 教学任务

“教学任务”是教师待办处理台，不是作业对象管理页。

第一版包含两类任务队列：

```text
待批改提交
未回答作业反馈
```

待批改提交任务展示：

- 课程名
- 作业名
- 学生姓名 / 学号 / 班级
- 提交时间
- 当前状态
- 进入批改操作

未回答作业反馈任务展示：

- 课程名
- 作业名
- 学生姓名 / 学号
- 问题或反馈内容摘要
- 发布时间
- 进入回答操作

教师从任务进入详情时，系统应自动带入课程、作业、学生提交和反馈上下文，不要求教师手动跨多个页面重新选择。

### 3.6 课程反馈任务

课程反馈任务用于课程整体反馈，不处理作业反馈。

任务来源：

1. 自动触发：课程状态变为“已结课”，或课程结束日期到达后，系统自动生成课程整体反馈任务。
2. 教师手动触发：课程进行中，教师可以手动发起课程整体反馈任务。

教师手动发起课程反馈任务时，第一版字段为：

- 反馈标题
- 反馈说明
- 开始时间
- 截止时间
- 反馈维度

默认反馈维度：

- 课程内容
- 教学方法
- 作业安排
- 授课节奏
- 学习收获
- 其他建议

第一版课程整体反馈不是复杂问卷。学生提交时填写反馈维度、反馈内容，并始终可以选择是否匿名。

教师查看课程反馈任务时，可看到：

- 提交人数 / 应提交人数
- 未提交学生名单
- 匿名反馈数 / 实名反馈数
- 各维度反馈数量
- 反馈明细列表

匿名规则：

- 教师可以查看哪些学生已提交、哪些学生未提交。
- 匿名只作用于反馈内容的身份展示。
- 匿名反馈在教师端显示为“匿名学生”。
- 教师不能把匿名反馈内容和具体学生身份关联起来。
- 教务员端保留真实提交人与反馈内容的审计能力。

任务生命周期：

```text
未开始
进行中
已截止
已归档
已取消
```

规则：

- `未开始`：教师可以修改标题、说明、开始时间、截止时间、反馈维度，也可以取消。
- `进行中`：学生可以提交反馈；教师可以延长截止时间、修改说明。
- `已截止`：学生不能再提交；教师查看汇总、统计和明细。
- `已归档`：教师确认已查看完毕，任务进入历史，不再作为待办提醒。
- `已取消`：任务被取消，学生端显示取消原因。

教师可以提前关闭进行中的反馈任务，提前关闭后状态变为“已截止”。如果已有学生提交，提前关闭需要填写原因；已有提交记录不能删除。

### 3.7 学生名单

学生名单是课程内学习状态视图，不是账号管理。

教师可以查看本课程学生，但不能增删学生账号，也不能禁用账号。

列表建议展示：

- 姓名
- 学号
- 班级
- 提交作业数 / 作业总数
- 缺交次数
- 已批改平均分
- 作业反馈数
- 课程反馈任务完成状态

点击学生后，可查看该学生在本课程内的学习详情：

- 作业提交记录
- 批改记录
- 作业反馈线程
- 课程整体反馈完成状态

匿名保护：

- 如果学生匿名提交课程整体反馈，教师在学生详情中只能看到“已提交匿名反馈”。
- 教师不能在学生详情中查看匿名反馈具体内容。
- 匿名反馈具体内容只出现在课程反馈任务汇总中，并显示为“匿名学生”。

基础成绩统计放在课程工作区和学生名单中，不单独设置“成绩分析”入口。

### 3.8 消息

教师端消息只做通知聚合和跳转，不承载业务编辑。

消息类型包括：

- 学生提交作业提醒
- 作业反馈待回答提醒
- 学生新增问题或反馈提醒
- 课程整体反馈任务已截止提醒
- 系统通知

消息状态第一版只区分：

```text
未读
已读
```

点击消息后跳转到对应业务位置：

- 提交提醒 -> 对应作业的提交批改页
- 作业反馈提醒 -> 对应反馈详情
- 课程整体反馈提醒 -> 对应反馈任务汇总
- 系统通知 -> 通知详情或工作台

任务是否已处理由业务对象本身决定，例如答案是否已批改、反馈是否已有回答、课程反馈任务是否已查看。

## 4. 教务员端

### 4.1 左侧导航

教务员端左侧导航建议保留以下入口：

```text
工作台
课程运营
用户管理
反馈审计
账号维护
```

入口职责：

| 入口 | 职责 |
| --- | --- |
| 工作台 | 展示平台教学运行概览、轻量异常提醒和最近新增课程。 |
| 课程运营 | 管理课程基础信息、课程状态、授课教师绑定，并查看课程运行概况。 |
| 用户管理 | 查看学生、教师、教务员账号；创建教师账号；禁用或恢复学生和教师账号。 |
| 反馈审计 | 审计课程整体反馈任务，查看匿名反馈真实提交人。 |
| 账号维护 | 教务员自己的资料、手机号、密码和安全操作。 |

教务员端不直接介入教师的作业批改和作业答疑流程。教务员可以通过课程运营或反馈审计查看关联信息，但不把“作业管理”“批改管理”“答疑管理”作为独立主入口。

### 4.2 工作台

教务员工作台用于展示教学运行总览。

建议展示：

- 课程总数
- 开课中课程数
- 已结课课程数
- 教师数
- 学生数
- 进行中的课程整体反馈任务
- 待关注异常提醒
- 最近新增课程

异常提醒第一版保持轻量，只读、可跳转，不自动处理。

异常提醒包括：

- 课程未分配教师
- 课程开始日期已到但状态仍为未开课
- 课程结束日期已过但状态仍为开课中
- 课程反馈任务提交率较低
- 被禁用账号数

“课程未分配教师”主要用于历史数据、导入数据或异常数据检查。正常创建课程时仍要求选择一个授课教师。

点击提醒后跳转到对应页面或筛选结果：

- 课程未分配教师 -> 课程运营列表
- 课程状态异常 -> 课程运营列表
- 反馈提交率低 -> 反馈审计任务详情
- 禁用账号 -> 用户管理

### 4.3 课程运营

课程运营是教务员端主入口，围绕课程基础信息和教学运行状态组织。

课程列表支持：

- 添加课程
- 按课程名称或课程代码搜索
- 按授课教师筛选
- 按学期筛选
- 按课程状态筛选
- 查看课程人数
- 查看授课教师
- 查看课程状态
- 进入课程详情

课程详情使用以下 Tab：

```text
课程概览 / 基础信息维护 / 授课与学生 / 作业概况 / 反馈审计
```

#### 4.3.1 课程概览

课程概览用于教务员查看课程运行状态。

主要内容：

- 课程名称
- 课程代码
- 学期
- 授课教师
- 上课时间与地点
- 课程状态
- 学生人数
- 作业数
- 课程反馈任务数
- 最近异常摘要

#### 4.3.2 基础信息维护

基础信息维护对应需求文档中的课程信息添加、查询、修改和删除能力。

课程字段包括：

- 课程名称
- 课程代码
- 授课教师
- 开课学期
- 课程描述
- 授课地点
- 上课时间
- 课程人数上限
- 开始日期
- 结束日期
- 课程状态

课程删除在 Web 端优先设计为“停止开课/归档”，不做常规物理删除。

设计解释：

- 需求获取文档中描述为“删除课程”，并提到删除后对应作业信息也会被删除。
- 从教学管理系统的正常使用逻辑看，直接物理删除课程会破坏历史作业、提交、批改和反馈记录。
- 因此 Web 第一版将“删除课程”解释为“停止开课/归档”，既满足课程不再对学生和教师开放的管理目标，也保留教学历史数据。

规则：

- 停止开课后，学生端不可再加入该课程。
- 停止开课后，教师端不可再为该课程发布新作业。
- 历史作业、提交、批改、作业反馈和课程反馈仍可查看。
- 如果未来必须做物理删除，应进入更高权限或二次审计流程，不作为第一版常规按钮。

#### 4.3.3 授课与学生

课程与教师关系第一版采用“一门课程一个授课教师”。

规则：

- 教务员创建课程时必须选择一个授课教师。
- 教务员修改课程时可以更换授课教师。
- 教师端只显示绑定给自己的授课课程。
- 更换授课教师后，新教师获得该课程的教师端访问权限。
- 旧教师不再出现在该课程的教师端课程列表中。
- 历史作业、批改、回答记录仍保留原作者信息，不因为更换授课教师而重写。

教务员可以查看课程学生名单，但第一版不管理选课关系。

学生名单规则：

- 教务员可以查看某门课程的学生名单。
- 教务员不手动添加学生到课程。
- 教务员不移除学生。
- 不做批量导入选课名单。
- 学生仍通过学生端加入课程。

#### 4.3.4 作业概况

作业概况只用于查看课程作业运行情况，不替代教师端作业管理。

建议展示：

- 作业列表
- 作业状态
- 发布时间
- 截止时间
- 提交人数
- 待批改人数
- 已批改人数

教务员不在该页面发布、修改或批改作业。

#### 4.3.5 反馈审计

课程详情中的反馈审计用于查看该课程下的课程整体反馈任务和反馈明细。

该 Tab 是全局“反馈审计”入口在单门课程内的局部视图。

### 4.4 用户管理

用户管理按账号类型分 Tab：

```text
学生账号 / 教师账号 / 教务员账号
```

#### 4.4.1 学生账号

学生账号由学生自己注册。

教务员可以：

- 查看学生列表
- 按姓名、学号、手机号搜索
- 按账号状态筛选
- 查看学生详情
- 禁用账号
- 恢复账号

教务员不直接修改学生个人资料，不重置学生密码。

#### 4.4.2 教师账号

教师账号由教务员创建。

教务员可以：

- 创建教师账号
- 查看教师列表
- 按姓名、工号、手机号搜索
- 按账号状态筛选
- 查看教师详情
- 禁用账号
- 恢复账号

教师忘记密码时，走手机号验证码找回流程。第一版不做教务员重置教师密码。

#### 4.4.3 教务员账号

教务员账号由系统管理员预置。

普通教务员端规则：

- 可以只读查看教务员账号列表。
- 不创建教务员账号。
- 不禁用其他教务员账号。
- 不重置教务员密码。

如需新增、禁用或恢复教务员账号，应由系统管理员在更高权限后台或运维流程中处理。

### 4.5 反馈审计

反馈审计只处理课程整体反馈任务，不处理作业反馈。

原因：

- 作业反馈属于教师与学生围绕作业批改的教学沟通。
- 课程整体反馈涉及匿名、课程质量和教学运行改进，更适合教务员审计。

反馈审计包含：

```text
反馈任务列表
反馈任务详情
反馈明细审计
```

反馈任务列表展示：

- 课程名
- 授课教师
- 反馈任务标题
- 任务来源：系统自动 / 教师手动
- 状态：未开始 / 进行中 / 已截止 / 已归档 / 已取消
- 提交人数 / 应提交人数
- 匿名反馈数
- 截止时间

反馈任务详情展示：

- 任务配置
- 提交统计
- 维度分布
- 未提交学生名单
- 反馈明细

反馈明细审计展示：

- 是否匿名
- 学生真实姓名
- 学号
- 班级
- 反馈维度
- 反馈内容
- 提交时间
- 教师端展示身份：实名 / 匿名学生

反馈审计第一版只读：

- 教务员可以查看匿名反馈的真实提交人。
- 教务员不编辑学生反馈。
- 教务员不删除学生反馈。
- 第一版不做审计备注。

### 4.6 账号维护

账号维护只处理当前教务员自己的账号信息。

可包含：

- 查看个人资料
- 修改个人资料
- 修改手机号
- 修改密码
- 注销或退出相关安全操作
