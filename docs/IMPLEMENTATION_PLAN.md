# 课程互动管理系统实施计划

## Phase 1. 文档与基线

- 完成需求抽取、冲突假设、需求追踪矩阵。
- 明确角色、权限、实体和状态机。
- 固化架构、API 和数据模型。

## Phase 2. 工程骨架

- 初始化 monorepo、工作区与共享配置。
- 建立 `apps/server`、`apps/web`、`apps/mobile`、`packages/*`。
- 配置 lint、typecheck、test、build 脚本。

## Phase 3. 后端基础

- 数据库初始化、迁移、种子数据。
- 配置层、日志层、错误处理中间件。
- 认证与会话管理。
- 用户、课程、选课、作业、提交、反馈、回复等核心模块。

## Phase 4. Web 客户端

- 视觉系统与应用壳层。
- 认证页、角色仪表盘。
- 学生课程/作业/提交流。
- 教师作业发布/批改/答疑流。
- 教务员课程管理与概览流。

## Phase 5. Mobile 客户端

- 与 Web 一致的角色导航与配色系统。
- 移动优先的课程、作业、提交与答疑页面。
- 支持下拉刷新、空状态、错误状态。

## Phase 6. 同步、测试与交付

- 完成 Query 失效、主动刷新、轮询同步策略。
- 编写单元测试、API 集成测试、关键流程测试。
- 输出 README、TEST_PLAN、TEST_REPORT、DEPLOYMENT。

## 当前优先级

1. 先完成工程骨架与后端核心数据流。
2. 再完成 Web 端核心闭环，保证演示主路径。
3. 最后补齐移动端、测试和交付文档。
